# EDT / WBS y Cronograma Inicial

## Estructura de trabajo

Para fines de planificación académica, el proyecto se divide en tres fases principales:

1. Fase inicial.
2. Fase de ejecución.
3. Fase de cierre.

## Fase 1: Inicial

### Actividades

- Revisión de documentación del museo.
- Levantamiento de requerimientos.
- Visita y validación de procesos.
- Reunión técnica sobre infraestructura.
- Elaboración del Project Charter.
- Elaboración de EDT/WBS.
- Elaboración de cronograma inicial.
- Identificación de riesgos y restricciones.

### Entregables

- Minutas.
- Documento de alcance preliminar.
- Project Charter.
- EDT/WBS.
- Cronograma inicial.

## Fase 2: Ejecución

### Actividades

- Consolidación de requerimientos funcionales.
- Definición de arquitectura tecnológica.
- Diseño UI/UX.
- Mockups de pantallas principales.
- Desarrollo del frontend público.
- Desarrollo de formulario de solicitudes educativas.
- Desarrollo del panel administrativo.
- Desarrollo o integración backend.
- Configuración de base de datos.
- Pruebas funcionales iniciales.

### Entregables

- Prototipo visual.
- Módulos principales desarrollados.
- Documentación técnica preliminar.
- Validaciones con stakeholders.

## Fase 3: Cierre

### Actividades

- Validación final de funcionalidades.
- Ajustes finales.
- Preparación de documentación.
- Presentación de resultados.
- Entrega final del proyecto.
- Firma o aprobación del supervisor.

### Entregables

- Versión final del prototipo/plataforma.
- Documentación final.
- Evidencias del proyecto.
- Informe de cierre.

## Observaciones

El cronograma debe mantenerse controlado debido a las limitaciones del período académico. Las funcionalidades de mayor complejidad deben priorizarse o dejarse para fases futuras.
